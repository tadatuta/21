# Gym Tracker - Telegram Mini App

Приложение для трекинга тренировок, разработанное как Telegram Mini App (TMA). Позволяет вести дневник упражнений, отслеживать прогресс и синхронизировать данные между устройствами.

## Функциональность

### Основные возможности
*   **Дневник тренировок**: Добавление подходов с указанием веса и повторений.
*   **Группировка**: Упражнения группируются по датам и типам.
*   **Типы тренировок**: Гибкое управление списком упражнений (добавление/удаление) через настройки.
*   **Быстрый ввод**: Возможность дублировать последний подход одной кнопкой.
*   **Редактирование**: Изменение и удаление сохраненных записей.

### Статистика
*   Просмотр общего объема (тоннаж) и количества повторений.
*   Фильтрация статистики по типам упражнений.
*   Графики прогресса максимального веса по датам.

### Синхронизация и хранение
*   **Облачное хранение**: Данные синхронизируются с Yandex Cloud Functions.
*   **Привязка к пользователю**: Используется Telegram User ID для идентификации данных.
*   **Offline-first**: При отсутствии интернета данные сохраняются локально и синхронизируются при появлении сети.
*   **Визуализация статуса**: Индикация процесса сохранения (Сохранение... / Сохранено / Ошибка).

## Технический стек

*   **Frontend**: TypeScript, Vite
*   **UI**: Vanilla CSS + DOM Manipulation (без использования тяжелых фреймворков типа React/Vue для максимальной производительности)
*   **Взаимодействие с Telegram**: Telegram Web App SDK (с моками для локальной разработки)
*   **Backend**: Yandex Cloud Functions (Serverless)
*   **Хранение**: LocalStorage + Remote API

## Установка и запуск

1.  **Установка зависимостей**:
    ```bash
    npm install
    ```

2.  **Запуск локального сервера разработки**:
    ```bash
    npm run dev
    ```
    Приложение будет доступно по адресу, указанному в консоли (обычно `http://localhost:5173`).
    Для локальной проверки логики Telegram используется мок-объект.

3.  **Сборка для продакшена**:
    ```bash
    npm run build
    ```

## Структура проекта

*   `src/main.ts` — Точка входа, роутинг и логика рендеринга страниц.
*   `src/storage/` — Сервис для работы с данными (LS + Sync).
*   `src/styles/` — Стили приложения.
*   `src/types/` — TypeScript интерфейсы.
*   `src/components/` — Вспомогательные UI компоненты.
*   `src/telegram-mock.ts` — Эмуляция Telegram API для разработки в браузере.

## Особенности реализации

*   Приложение реализует SPA-архитектуру на чистом JS/TS.
*   Использует CSS Variables для темизации (поддержка темной темы Telegram).
*   Адаптировано под мобильные устройства.
